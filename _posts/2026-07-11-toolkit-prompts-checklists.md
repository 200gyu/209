## ✍️ 프롬프트 템플릿

### 1. 운영 지표 요약 (경영진용)
```text
너는 서비스 운영 분석가야. 아래 지표를 경영진용 3줄 요약으로 정리해줘.
- 규칙: 전주 대비 증감률과 '주목할 이슈'를 반드시 포함
- 톤: 담백하게, 과장 금지
- 형식: 불릿 3개, 각 줄 앞에 📈 / 📉 / ⚠️ 이모지
[데이터]
가입: (이번주) / (전주)
이탈: (이번주) / (전주)
전환율: (이번주) / (전주)
```

### 2. VOC(고객 문의) 분류
```text
너는 10년 차 CS 운영 매니저야. 아래 문의들을 분류해줘.
- 카테고리: [환불 / 버그 / 기능요청 / 사용법 / 기타]
- 먼저 근거를 한 줄 적고, 표로 정리: 문의 | 카테고리 | 근거
- 예) "결제가 안 돼요" → 버그
- 예) "다크모드 있으면 좋겠어요" → 기능요청
[문의 목록]
- ...
```

### 3. 회의록 → 액션 아이템 추출
```text
아래 회의 메모에서 '결정사항'과 '액션 아이템'만 뽑아줘.
- 액션 아이템은 [담당자 추정 / 기한 언급]을 함께 표로
- 애매하면 '확인 필요'라고 표시 (지어내지 말 것)
[회의 메모]
...
```

> ⚠️ **환각 방지 팁**: 프롬프트에 항상 *"모르면 지어내지 말고 '확인 필요'라고 답해"* 를 넣으세요.
{: .prompt-warning }

## ✅ 에이전트 기획 체크리스트

- [ ] 이 업무를 **입력 → 처리 → 출력**으로 쪼갰는가?
- [ ] 각 단계를 `🟢자동 / 🟡AI보조 / 🔴사람판단`으로 태깅했는가?
- [ ] 필요한 **도구(tool)** 를 목록으로 적었는가?
- [ ] **MVP 범위**를 "제일 아픈 것 하나"로 좁혔는가?
- [ ] MVP는 **API로 빠르게** 시작하는가? (고정비 리스크 회피)
- [ ] **사람 검수 단계**를 넣었는가? (환각 대비)
- [ ] 성공을 잴 **지표**가 있는가? (예: 작업시간 절감률)

## ✅ 프롬프트 작성 체크리스트

- [ ] **역할**을 부여했는가? ("너는 ○○ 전문가야")
- [ ] **맥락**(왜/누구를 위해)을 줬는가?
- [ ] **예시** 2~3개를 넣었는가?
- [ ] **출력 형식**을 못박았는가? (표/불릿/JSON 등)
- [ ] 어려운 판단은 **단계적으로 생각**하게 했는가?
- [ ] 여러 입력으로 **테스트**하고 실패 케이스를 보강했는가?
