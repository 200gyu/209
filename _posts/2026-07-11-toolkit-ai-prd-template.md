> 일반 기획서에 **AI 기능 특유의 항목**(무엇을 AI가/사람이 판단할지, 환각 대비, 데이터 출처·권한)을 더한 PRD 템플릿입니다. 복사해서 채우세요.
{: .prompt-tip }

## 📋 PRD 템플릿 (복사용)

```markdown
# [기능명] PRD

## 1. 문제 (Problem)
- 누가, 어떤 상황에서, 무엇이 불편한가:
- 지금은 어떻게 해결하고 있나(현행):

## 2. 목표 & 성공 지표 (Goal / Metric)
- 이 기능이 성공했다고 말할 수 있는 숫자:
  - 예) 작업 시간 __분 → __분, 정확도 __%, 초안 활용률 __%

## 3. 대상 사용자 (User)
- 핵심 사용자 / 사용 빈도:

## 4. 해결 방식 (Solution)
- 사용자 흐름(입력 → 처리 → 출력):
- 화면/기능 요소:

## 5. AI 사용 범위 (AI Scope)  ⭐AI 특유
- AI가 판단하는 것:
- 사람이 판단·검수하는 것:
- 사용할 모델/방식(LLM · RAG · 규칙):

## 6. 데이터 (Data)  ⭐AI 특유
- 입력 데이터 출처:
- 민감/개인정보 여부 & 처리 권한:
- 학습/저장 여부:

## 7. 리스크 & 대응 (Risk)  ⭐AI 특유
- 환각(잘못된 생성) 대응:
- 실패/빈 응답 시 동작:
- 비용(호출당 과금) 통제:

## 8. 범위 (Scope)
- MVP에 포함:
- 이번엔 제외(나중에):

## 9. 오픈 이슈 (Open Questions)
- 아직 확인이 필요한 것:
```

## ✅ AI 기능 기획 체크리스트

- [ ] **AI/사람 판단 경계**를 명확히 나눴는가? (5번)
- [ ] **성공 지표가 숫자**인가? (2번)
- [ ] **데이터 권한·프라이버시**를 확인했는가? (6번)
- [ ] **환각·실패 시 동작**을 정의했는가? (7번)
- [ ] **사람 검수 단계**가 흐름에 있는가?
- [ ] **MVP 범위**를 "제일 아픈 것 하나"로 좁혔는가?
- [ ] **비용**(API 과금)을 추정했는가?

## 🧭 왜 'AI 특유' 항목이 중요한가

일반 기능과 달리 AI 기능은 **틀릴 수 있고(환각), 돈이 들며(호출 과금), 데이터에 민감**합니다. 이 세 가지를 기획 단계에서 못박지 않으면 나중에 크게 물립니다. (자세한 이유는 [에이전트 vs RPA]({% post_url 2026-07-11-ai-agent-vs-rpa %})·[RAG]({% post_url 2026-07-11-rag-explained %}) 글 참고)
